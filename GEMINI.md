# Gemini CLI 작업 지침 (moraimpc_ros1)

사용자는 이 프로젝트의 모든 공학적 작업(파일 수정, 명령어 실행, 삭제 등)에 대해 전권을 위임하였습니다.

## 핵심 원칙
- **자율성 (Autonomy)**: 모든 작업 수행 전 "수정해도 될까요?" 또는 "실행할까요?"와 같은 확인 질문을 생략하고 즉시 실행합니다.
- **사전 분석 필수**: 코드 변경 전 반드시 기존 코드의 구조와 의존성, 그리고 주행 로그(`mpc_log.json`)를 철저히 분석합니다.
- **안정성 우선**: MPC 제어 시 진동을 최소화하고 안정적인 주행을 최우선으로 합니다. 모델링 미스매치와 제어 지연을 고려하여 파라미터를 튜닝합니다.

## 기술적 지침
- **곡률 이득 (kappa_gain)**: MORAI 차량의 강력한 조향 응답성(약 6배)을 모델에 상시 반영합니다.
- **제약 조건 동기화**: MPC의 입력 제약(`u_min`, `u_max`)은 실제 조향 변화율 제한(`max_steer_rate_`)과 반드시 동기화되어야 합니다.
- **로그 모니터링**: 주행 후 생성되는 JSON 로그를 분석하여 제어 오차와 OSQP 연산 시간을 상시 모니터링합니다.
